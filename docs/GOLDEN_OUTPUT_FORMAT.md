# GOLDEN OUTPUT FORMAT — รูปแบบไฟล์จริงที่ทำเสร็จแล้ว (อ้างอิงหลัก ห้ามมั่ว)

เอกสารนี้ถอดจาก **ไฟล์ผลลัพธ์จริง 2 ไฟล์** ที่ทำเสร็จด้วยมือหลังได้ข้อมูลดิบ
(แบบประเมินแผนกปกครอง วพอ.พอ. ปีการศึกษา 2566) เป็น **มาตรฐานเดียวที่ต้องยึด**
เมื่อระบบสร้าง Excel/PDF ห้ามเดา/ห้ามสร้างรูปแบบใหม่ที่ไม่ตรงเอกสารนี้

> หลักคิดของเจ้าของงาน: **ถ้าจัด Excel ให้ดี PDF ก็แทบไม่ต่าง** — เพราะ PDF = พิมพ์ sheet รายงานออกมาตรง ๆ

---

## 0) ภาพรวม

- **1 workbook ต่อ 1 งาน/1 กิจกรรมใหญ่** (เช่น กิจกรรมส่งเสริมสุขภาพ, โครงการส่งเสริม...)
- **แต่ละกิจกรรมย่อย/ชมรม = 2 sheet ที่คู่กัน**
  1. `(ชมรมX)` — **sheet ข้อมูลดิบ** (input ที่กรอกคะแนน)
  2. `X` — **sheet รายงาน** (output จัดรูปสำหรับพิมพ์)
- ตัวอย่าง: `(ชมรมวอลเลย์บอล)` + `วอลเลย์บอล`, `(ชมรมดนตรีไทย)` + `ดนตรีไทย`
- **รายงานเป็นราย "กิจกรรม/ชมรม" รวมทุกชั้นปีเข้าด้วยกัน** (หัวเรื่องระบุ "นพอ.ชั้นปีที่ 1-4")
  → **ไม่ต้องแยกรายงานเป็นราย ปี1/ปี2/ปี3/ปี4** เว้นแต่เจ้าของงานสั่ง

---

## 1) Sheet ข้อมูลดิบ `(ชมรมX)`

| แถว | เนื้อหา |
|---|---|
| แถว 1 | `แบบประเมินแผนกปกครอง` (หัวเรื่องดิบ) |
| **แถว 2 = HEADER** | `ลำดับ` \| `ชื่อ - สกุล` \| (คอลัมน์ต่อของชื่อ) \| `นพอ.ชั้นปีที่` \| `ข้อ1` \| `2` \| `3` \| … \| `n` \| `X` \| `SD` |
| แถว 3..N | 1 แถว = 1 ผู้ตอบ |
| แถวท้าย | `ค่าเฉลี่ย` (ค่าเฉลี่ยรายข้อ), `SD` (SD รายข้อ) |

**กติกาสำคัญของ sheet ดิบ:**
- **HEADER อยู่แถวที่ 2 ไม่ใช่แถวที่ 1** (แถว 1 เป็นหัวเรื่อง)
- **ยศ/คำนำหน้าอยู่คนละคอลัมน์กับชื่อ**: คอลัมน์ B = `นพอ.` (ยศ), คอลัมน์ C = ชื่อ-นามสกุล
  - บางเซลล์ชื่อมี `นพอ.` ซ้ำนำหน้าอยู่แล้ว → clean รวมเป็น "นพอ. + ชื่อสกุล" ไม่ตัดตัวอักษรผิดช่อง
- คอลัมน์ `นพอ.ชั้นปีที่` (คอลัมน์ D) = ชั้นปี (ค่าตัวอย่าง 4)
- ข้อประเมินเป็น **ตัวเลข `ข้อ1, 2, 3, …, n`** (ไม่มีข้อความคำถามใน sheet ดิบ)
- ท้ายคอลัมน์ข้อ = `X` และ `SD` **รายบุคคล** (ต่อคน)
- แถวล่างสุด = `ค่าเฉลี่ย` และ `SD` **รายข้อ** (ต่อข้อ)
- คะแนน 1–5 เท่านั้น

## 2) Sheet รายงาน `X` (สำคัญสุด — นี่คือสิ่งที่พิมพ์เป็น PDF)

| แถว | เนื้อหา (verbatim) |
|---|---|
| แถว 1 | `วิทยาลัยพยาบาลทหารอากาศ` |
| แถว 2 | ชื่อแบบประเมิน + (ชมรมX) + `นพอ.ชั้นปีที่ 1-4 ปีการศึกษา 2566` |
| แถว 3 | `เกณฑ์การประเมิน` |
| แถว 4 | `4.51 – 5.00  หมายถึง  นักเรียนมีความเห็นด้วยในระดับ  มากที่สุด` |
| แถว 5 | `3.51 – 4.50  หมายถึง  นักเรียนมีความเห็นด้วยในระดับ  มาก` |
| แถว 6 | `2.51 – 3.50  หมายถึง  นักเรียนมีความเห็นด้วยในระดับ  ปานกลาง` |
| แถว 7 | `1.51 – 2.50  หมายถึง  นักเรียนมีความเห็นด้วยในระดับ  น้อย` |
| แถว 8 | `1.00 – 1.50  หมายถึง  นักเรียนมีความเห็นด้วยในระดับ  น้อยที่สุด/ ไม่เห็นด้วย` |
| แถว 9 | หัวคอลัมน์กลุ่ม `ประเมิน` |
| แถว 10 | หัวตาราง `X` \| `SD` |
| แถว 11..n | **รายข้อ**: คอลัมน์ A = **ข้อความคำถามเต็ม** (เช่น `1. การร่วมกิจกรรมกีฬา...`), คอลัมน์ X, คอลัมน์ SD |
| แถวรวม | `รวม` + X รวม + SD รวม |
| แถวสรุป | `ผลการประเมินความพึงพอใจในการเรียนการจัดกิจกรรม (นักเรียน)` |
| แถวสรุป+1 | `ค่าเฉลี่ย 3.51 ขึ้นไป คิดเป็นร้อยละ NN.NN` |

**กติกาสำคัญของ sheet รายงาน:**
- **ข้อความคำถามจริงอยู่ที่นี่** (คอลัมน์ A/B ของ sheet รายงาน) — ไม่ใช่ใน sheet ดิบ
  - การจับคู่ `ข้อN → ข้อความคำถาม` เป็น **template คงที่ต่อประเภทแบบประเมิน**
  - ห้ามใช้ `Q1/ข้อ1` เป็นชื่อข้อในรายงานสุดท้าย — ต้องเป็นข้อความเต็ม
- X, SD ทศนิยม (ในไฟล์จริงเก็บเต็ม แต่ **แสดง 2 ตำแหน่ง**)
- มีแถว `รวม` (ค่าเฉลี่ย/SD รวมทุกข้อ)
- ปิดท้ายด้วย **ร้อยละความพึงพอใจ** = สัดส่วนผู้ตอบที่มีค่าเฉลี่ย ≥ 3.51 (คิดเป็น %)

## 3) รูปแบบย่อยที่พบ (variation)

ไฟล์ 2 (โครงการส่งเสริม...) มี **การจัดกลุ่มมิติ (dimension)** เพิ่ม:
- คอลัมน์ A = ชื่อ **ด้าน/มิติ** (เช่น `ทักษะการใช้ชีวิตและ...`)
- คอลัมน์ B = **ข้อความคำถามรายข้อ** (`1. สมาชิกในชมรมมีความ...`)
- คอลัมน์ X = J, คอลัมน์ SD = K (เลื่อนขวา 1 คอลัมน์)

ระบบต้องรองรับทั้ง 2 แบบ:
- **แบบ A** (กีฬา): ข้อความคำถามอยู่คอลัมน์ A, X=I, SD=J
- **แบบ B** (มีมิติ): มิติคอลัมน์ A, ข้อความคำถามคอลัมน์ B, X=J, SD=K

## 4) สิ่งที่ระบบปัจจุบันทำ "เกิน/ไม่ตรง" golden (ต้องปรับ)

```text
[ ] ระบบแยกรายงานราย ปี1-4 → golden ไม่แยก (รายงานราย "กิจกรรม/ชมรม" รวมทุกปี)
[ ] ระบบมี Cover/Executive/Dashboard เยอะ → golden มีแค่ 2 sheet ต่อกิจกรรม (ดิบ + รายงาน)
[ ] ระบบต้องได้ข้อความคำถามจาก template (map ข้อN→ข้อความ) ไม่ใช่หวังจากหัวคอลัมน์ดิบ
[ ] ระบบต้อง clean ยศ (นพอ.) ที่อยู่คนละคอลัมน์กับชื่อ และที่ซ้ำนำหน้าชื่อ
[ ] ระบบต้องอ่าน HEADER ที่แถว 2 (แถว 1 เป็นหัวเรื่อง)
[ ] รายงานต้องปิดท้ายด้วย "ร้อยละความพึงพอใจ (ค่าเฉลี่ย ≥ 3.51)"
```

## 5) เกณฑ์ตรวจรับ (เทียบ golden นี้)

```text
[ ] มี sheet ดิบ (ชมรมX) + sheet รายงาน X คู่กันต่อกิจกรรม
[ ] header ดิบอ่านจากแถว 2, ยศแยกคอลัมน์, ชั้นปีอ่านถูก
[ ] รายงานมีหน่วยงาน + ชื่อแบบประเมิน + เกณฑ์ 5 ระดับ (ข้อความตรง §2)
[ ] ตารางรายข้อใช้ "ข้อความคำถามจริง" + X + SD (2 ตำแหน่ง)
[ ] มีแถว "รวม"
[ ] มีบรรทัดสรุป "ร้อยละความพึงพอใจ ค่าเฉลี่ย 3.51 ขึ้นไป"
[ ] คะแนนอยู่ในช่วง 1–5 ไม่มีสูตร error
```
