# GOLDEN OUTPUT FORMAT — รูปแบบไฟล์จริงที่ทำเสร็จแล้ว (อ้างอิงหลัก ห้ามมั่ว)

เอกสารนี้ถอดจาก **ไฟล์ผลลัพธ์จริง 2 ไฟล์** ที่ทำเสร็จด้วยมือหลังได้ข้อมูลดิบ
(แบบประเมินแผนกปกครอง วพอ.พอ. ปีการศึกษา 2566) เป็น **มาตรฐานเดียวที่ต้องยึด**
เมื่อระบบสร้าง Excel/PDF ห้ามเดา/ห้ามสร้างรูปแบบใหม่ที่ไม่ตรงเอกสารนี้

> หลักคิดของเจ้าของงาน: **ถ้าจัด Excel ให้ดี PDF ก็แทบไม่ต่าง** — เพราะ PDF = พิมพ์ sheet รายงานออกมาตรง ๆ

---

## 0) ภาพรวม

- **1 workbook ต่อ 1 งาน/1 กิจกรรมใหญ่** (เช่น กิจกรรมส่งเสริมสุขภาพ, โครงการส่งเสริม...)
- **แต่ละกิจกรรมย่อย/ชมรม = 2 sheet ที่คู่กัน**
  1. `(ชมรมX)` — **sheet ข้อมูลดิบ** (input ที่กรอกคะแนน)
  2. `X` — **sheet รายงาน** (output จัดรูปสำหรับพิมพ์)
- ตัวอย่าง: `(ชมรมวอลเลย์บอล)` + `วอลเลย์บอล`, `(ชมรมดนตรีไทย)` + `ดนตรีไทย`
- **รายงานเป็นราย "กิจกรรม/ชมรม" รวมทุกชั้นปีเข้าด้วยกัน** (หัวเรื่องระบุ "นพอ.ชั้นปีที่ 1-4")
  → **ไม่ต้องแยกรายงานเป็นราย ปี1/ปี2/ปี3/ปี4** เว้นแต่เจ้าของงานสั่ง

## 1) Sheet ข้อมูลดิบ `(ชมรมX)`

| แถว | เนื้อหา |
|---|---|
| แถว 1 | `แบบประเมินแผนกปกครอง` (หัวเรื่องดิบ) |
| **แถว 2 = HEADER** | `ลำดับ` \| `ชื่อ - สกุล` \| (คอลัมน์ต่อของชื่อ) \| `นพอ.ชั้นปีที่` \| `ข้อ1` \| `2` \| `3` \| … \| `n` \| `X` \| `SD` |
| แถว 3..N | 1 แถว = 1 ผู้ตอบ |
| แถวท้าย | `ค่าเฉลี่ย` (ค่าเฉลี่ยรายข้อ), `SD` (SD รายข้อ) |

**กติกาสำคัญของ sheet ดิบ:**
- **HEADER อยู่แถวที่ 2 ไม่ใช่แถวที่ 1** (แถว 1 เป็นหัวเรื่อง)
- **ยศ/คำนำหน้าอยู่คนละคอลัมน์กับชื่อ**: คอลัมน์ B = `นพอ.` (ยศ), คอลัมน์ C = ชื่อ-นามสกุล
  - บางเซลล์ชื่อมี `นพอ.` ซ้ำนำหน้าอยู่แล้ว → clean รวมเป็น "นพอ. + ชื่อสกุล" ไม่ตัดตัวอักษรผิดช่อง
- คอลัมน์ `นพอ.ชั้นปีที่` (คอลัมน์ D) = ชั้นปี (ค่าตัวอย่าง 4)
- ข้อประเมินเป็น **ตัวเลข `ข้อ1, 2, 3, …, n`** (ไม่มีข้อความคำถามใน sheet ดิบ)
- ท้ายคอลัมน์ข้อ = `X` และ `SD` **รายบุคคล** (ต่อคน)
- แถวล่างสุด = `ค่าเฉลี่ย` และ `SD` **รายข้อ** (ต่อข้อ)
- คะแนน 1–5 เท่านั้น

## 2) Sheet รายงาน `X` (สำคัญสุด — นี่คือสิ่งที่พิมพ์เป็น PDF)

| แถว | เนื้อหา (verbatim) |
|---|---|
| แถว 1 | `วิทยาลัยพยาบาลทหารอากาศ` |
| แถว 2 | ชื่อแบบประเมิน + (ชมรมX) + `นพอ.ชั้นปีที่ 1-4 ปีการศึกษา 2566` |
| แถว 3 | `เกณฑ์การประเมิน` |
| แถว 4 | `4.51 – 5.00  หมายถึง  นักเรียนมีความเห็นด้วยในระดับ  มากที่สุด` |
| แถว 5 | `3.51 – 4.50  หมายถึง  นักเรียนมีความเห็นด้วยในระดับ  มาก` |
| แถว 6 | `2.51 – 3.50  หมายถึง  นักเรียนมีความเห็นด้วยในระดับ  ปานกลาง` |
| แถว 7 | `1.51 – 2.50  หมายถึง  นักเรียนมีความเห็นด้วยในระดับ  น้อย` |
| แถว 8 | `1.00 – 1.50  หมายถึง  นักเรียนมีความเห็นด้วยในระดับ  น้อยที่สุด/ ไม่เห็นด้วย` |
| แถว 9 | หัวคอลัมน์กลุ่ม `ประเมิน` |
| แถว 10 | หัวตาราง `X` \| `SD` |
| แถว 11..n | **รายข้อ**: คอลัมน์ A = **ข้อความคำถามเต็ม** (เช่น `1. การร่วมกิจกรรมกีฬา...`), คอลัมน์ X, คอลัมน์ SD |
| แถวรวม | `รวม` + X รวม + SD รวม |
| แถวสรุป | `ผลการประเมินความพึงพอใจในการเรียนการจัดกิจกรรม (นักเรียน)` |
| แถวสรุป+1 | `ค่าเฉลี่ย 3.51 ขึ้นไป คิดเป็นร้อยละ NN.NN` |

**กติกาสำคัญของ sheet รายงาน:**
- **ข้อความคำถามจริงอยู่ที่นี่** (คอลัมน์ A/B ของ sheet รายงาน) — ไม่ใช่ใน sheet ดิบ
- ⭐ **แหล่งข้อความคำถาม = อ่านจาก sheet รายงาน `X` ที่มากับไฟล์ที่อัปโหลด** (ไฟล์ดิบมี sheet รายงานคู่มาให้แล้ว)
  - ระบบ **ไม่ต้องเก็บ template ข้อความเอง** และ **ไม่ต้องเดา** — จับคู่ sheet ดิบ `(ชมรมX)` กับ sheet รายงาน `X` ในไฟล์เดียวกัน
  - ขั้นตอน: อ่าน **ข้อความคำถาม + เกณฑ์** จาก sheet รายงาน, อ่าน **คะแนนรายคน** จาก sheet ดิบ, คำนวณ X/SD, แล้วเติมผลลงตารางรายงาน
  - ถ้าไฟล์ไม่มี sheet รายงานคู่ (มีแต่ ข้อ1..n) → QA = REVIEW เพราะอ่านข้อความคำถามไม่ได้
  - ห้ามใช้ `Q1/ข้อ1` เป็นชื่อข้อในรายงานสุดท้าย — ต้องเป็นข้อความเต็ม
- X, SD ทศนิยม (ในไฟล์จริงเก็บเต็ม แต่ **แสดง 2 ตำแหน่ง**)
- มีแถว `รวม` (ค่าเฉลี่ย/SD รวมทุกข้อ)
- ปิดท้ายด้วย **ร้อยละความพึงพอใจ** = สัดส่วนผู้ตอบที่มีค่าเฉลี่ย ≥ 3.51 (คิดเป็น %)

## 3) รูปแบบย่อยที่พบ (variation)

ไฟล์ 2 (โครงการส่งเสริม...) มี **การจัดกลุ่มมิติ (dimension)** เพิ่ม:
- คอลัมน์ A = ชื่อ **ด้าน/มิติ** (เช่น `ทักษะการใช้ชีวิตและ...`)
- คอลัมน์ B = **ข้อความคำถามรายข้อ** (`1. สมาชิกในชมรมมีความ...`)
- คอลัมน์ X = J, คอลัมน์ SD = K (เลื่อนขวา 1 คอลัมน์)

ระบบต้องรองรับทั้ง 2 แบบ:
- **แบบ A** (กีฬา): ข้อความคำถามอยู่คอลัมน์ A, X=I, SD=J
- **แบบ B** (มีมิติ): มิติคอลัมน์ A, ข้อความคำถามคอลัมน์ B, X=J, SD=K

## 4) สิ่งที่ระบบปัจจุบันทำ "เกิน/ไม่ตรง" golden (ต้องปรับ)

```text
[ ] ระบบแยกรายงานราย ปี1-4 → golden ไม่แยก (รายงานราย "กิจกรรม/ชมรม" รวมทุกปี)
[ ] ระบบมี Cover/Executive/Dashboard เยอะ → golden มีแค่ 2 sheet ต่อกิจกรรม (ดิบ + รายงาน)
[ ] ระบบต้องอ่านข้อความคำถามจาก "sheet รายงาน X" ที่มากับไฟล์ (จับคู่กับ sheet ดิบ (ชมรมX)) ไม่ใช่หวังจากหัวคอลัมน์ดิบ
[ ] ระบบต้อง clean ยศ (นพอ.) ที่อยู่คนละคอลัมน์กับชื่อ และที่ซ้ำนำหน้าชื่อ
[ ] ระบบต้องอ่าน HEADER ที่แถว 2 (แถว 1 เป็นหัวเรื่อง)
[ ] รายงานต้องปิดท้ายด้วย "ร้อยละความพึงพอใจ (ค่าเฉลี่ย ≥ 3.51)"
```

## 5) เกณฑ์ตรวจรับ (เทียบ golden นี้)

```text
[ ] มี sheet ดิบ (ชมรมX) + sheet รายงาน X คู่กันต่อกิจกรรม
[ ] header ดิบอ่านจากแถว 2, ยศแยกคอลัมน์, ชั้นปีอ่านถูก
[ ] รายงานมีหน่วยงาน + ชื่อแบบประเมิน + เกณฑ์ 5 ระดับ (ข้อความตรง §2)
[ ] ตารางรายข้อใช้ "ข้อความคำถามจริง" + X + SD (2 ตำแหน่ง)
[ ] มีแถว "รวม"
[ ] มีบรรทัดสรุป "ร้อยละความพึงพอใจ ค่าเฉลี่ย 3.51 ขึ้นไป"
[ ] คะแนนอยู่ในช่วง 1–5 ไม่มีสูตร error
```

## 6) ประเภทที่ 2 — ประเมินผู้สอนรายบุคคล / รายวิชา (ทหาร)

ประเภทนี้ประเมิน **ผู้สอน (อาจารย์) รายบุคคล ต่อรายวิชา** ประกอบด้วย **3 workbook สัมพันธ์กัน**:

### 6.1 ไฟล์ดิบ (แบบประเมินรายวิชา)
- Sheet แรก `แบบประเมินรายวิชา` (รวม) + **1 sheet ต่อ 1 ผู้สอน** (ชื่อ sheet = ชื่ออาจารย์ เช่น `น.ท.ธงชัย เรืองภักดี`)
- แต่ละ sheet ผู้สอน: header แถว 2 = `เลขที่` \| (ชื่อ นพอ. col B) \| `ข้อ1`..`ข้อ10` — แต่ละแถว = นักเรียน 1 คนให้คะแนนอาจารย์คนนั้น
- ยศ/คำนำหน้าอยู่กับชื่อ (นพอ.) — clean เหมือน §1

### 6.2 ไฟล์รายงาน (ผลการประเมินความพึงพอใจ)
- Sheet `ผลการประเมินความพึงพอใจ` (ภาพรวมรายวิชา, ~14 ข้อ) + **1 sheet ต่อผู้สอน** (~12 ข้อ)
- **Sheet รายผู้สอน** layout:
  - r1: `ผลการประเมินความพึงพอใจในการเรียนการจัดการเรียนการสอน...`
  - r2: `ผู้สอนรายบุคคล วิชาการฝึกทหาร N ปีการศึกษา ...`
  - r3: `ชื่ออาจารย์ <ยศ ชื่อ-สกุล>`
  - r4: `เกณฑ์การประเมิน` + r5-9: เกณฑ์ 5 ระดับ (ข้อความเดียวกับ §2)
  - r11: `ข้อมูลเกี่ยวกับการเรียนการสอน` \| `x` \| `SD`
  - r12..: **ข้อประเมินรายข้อพร้อมข้อความคำถามเต็ม** (คอลัมน์ A) \| x (B) \| SD (C)
  - แถว `ค่าเฉลี่ยรวม` \| x \| SD
  - แถวสรุป + `ทราบแล้ว` (การรับทราบ) + `แนวทางการพัฒนา`
- **Sheet ภาพรวมรายวิชา**: เหมือน §2 (เกณฑ์ + ตาราง 14 ข้อ + `รวม` + `ค่าเฉลี่ย 3.51 ขึ้นไป คิดเป็นร้อยละ NN` + `ข้อเสนอแนะ`)

### 6.3 ไฟล์สรุปรวม (ผลการประเมินผู้สอน — แยกตามชั้นปี)
- Sheet `ทหาร 1`..`ทหาร 4` (= ชั้นปี 1-4)
- แต่ละ sheet = **1 รายวิชา**: หัว `ผลการประเมินผู้สอนรายวิชา/ การปรับปรุง/ พัฒนา`
  - ตาราง: `รายวิชา` \| `ผู้สอน` \| `ผลการประเมิน (X \| SD)`
  - คอลัมน์รายวิชา merge (เช่น `พอวท 1201 การฝึกทหาร 1`), ไล่ผู้สอนทีละคนพร้อม X/SD
  - ปิดท้าย `ค่าเฉลี่ย` (X รวมของวิชา) + `SD`

> ประเภทนี้ **แยกตามชั้นปี/รายวิชาได้** (พอวท 1201=ปี1 ... 4205=ปี4) ต่างจากประเภทกิจกรรม/ชมรม (§0-5) ที่รวมทุกชั้นปี

## 6ก) ประเภทที่ 3 — ประเมินอัตลักษณ์ (งาม / สง่า)

ประเมิน **อัตลักษณ์นักเรียน** วัด 2 มิติ (**งาม** และ **สง่า**) รวม ~30 ข้อ

### โครงสร้างไฟล์
- **แยกรายงานราย ชั้นปี**: `อัตลักษณ์` (ภาพรวม) + `อัตลักษณ์ ปี1`..`ปี4`
- Sheet ดิบราย ชั้นปี: `ปี 1`..`ปี4` (header แถว 2: `ลำดับที่` \| ยศ(B) \| ชื่อ(C) \| อีเมล(D) \| `1`,`2`,`3`… เป็นเลข)
- Sheet **`ข้อคำถาม`** = เก็บข้อความคำถามเต็ม 30 ข้อ (แหล่งข้อความคำถาม)
- มี sheet เปรียบเทียบข้ามปีการศึกษา (`งาม สง่า ปีการศึกษา 2567/2566/2565...`)

### ⭐ ตารางรายข้อ = **หลายมิติ (multi-dimension)**
```text
r9 : ข้อมูลเกี่ยวกับการเรียน | ประเมิน งาม (X,SD) | ประเมิน สง่า (X,SD) | โดยรวม | ระดับ
r10:                          | X | SD             | X | SD              |        |
r11..: <ข้อความคำถามข้อ N>    | (เติมเฉพาะมิติของข้อนั้น)                 | โดยรวม | ระดับ
```
- แต่ละข้อ **สังกัดมิติเดียว** (บางข้อเป็น "งาม" เติม X/SD ที่คอลัมน์งาม, บางข้อเป็น "สง่า" เติมคอลัมน์สง่า)
- คอลัมน์ `โดยรวม` + `ระดับ` (มากที่สุด/มาก/…)
- Sheet `ความพึงพอใจ รวม` ใช้แบบ **มิติเดียว** (X \| SD) สำหรับสรุปพึงพอใจ

> ประเภทนี้ **แยกราย ชั้นปี** + **ตารางหลายมิติ** — ระบบต้องรองรับตารางที่มีหลายชุด X/SD ต่อข้อ

## 7) Building blocks ที่ใช้ซ้ำทุกประเภท (มาตรฐานร่วม)

ทุก sheet รายงานใช้บล็อกเดียวกันนี้ (คงข้อความเป๊ะ):

1. **หัว**: `วิทยาลัยพยาบาลทหารอากาศ` + ชื่อแบบประเมิน
2. **เกณฑ์การประเมิน 5 ระดับ** (ข้อความตรงตาม §2 r4-8)
3. **ตารางรายข้อ**: `ข้อความคำถามเต็ม` \| `X` \| `SD` (ทศนิยม 2 ตำแหน่ง)
   - รองรับได้ทั้ง **มิติเดียว** (X\|SD) และ **หลายมิติ** (เช่น งาม X/SD \| สง่า X/SD \| โดยรวม \| ระดับ ตาม §6ก)
   - **สูตรที่ตรวจสอบแล้ว (ตรง golden 100%)**: `X = ค่าเฉลี่ย (AVERAGE)`, `SD = STDEV.S (sample SD)` — ไม่ใช่ population SD
4. **แถวรวม**: `รวม` / `ค่าเฉลี่ยรวม` + X + SD
5. **สรุปพึงพอใจ**: `ค่าเฉลี่ย 3.51 ขึ้นไป คิดเป็นร้อยละ NN`
   - **สูตรที่ตรวจสอบแล้ว**: ร้อยละ = (จำนวนผู้ตอบที่ **ค่าเฉลี่ยรายบุคคล ≥ 3.51**) ÷ (ผู้ตอบทั้งหมด) × 100
6. ข้อความคำถาม/เกณฑ์ **อ่านจาก sheet รายงานในไฟล์เสมอ** (ไม่เก็บ template เอง ไม่เดา)
7. **การปัดเลข**: ทุกค่า X / SD / ร้อยละ **แสดงทศนิยม 2 ตำแหน่ง ด้วย ROUND** (ปัดครึ่งขึ้น) — ตรงกับ `CONFIG.ROUND_MODE = 'ROUND'`

ระบบต้อง **จำแนกประเภท** จากโครงสร้าง sheet:
- คู่ `(ชมรมX)`+`X` → กิจกรรม/ชมรม (§1-5)
- sheet ต่อชื่ออาจารย์ + `แบบประเมินรายวิชา`/`ผลการประเมินความพึงพอใจ` → ผู้สอนรายวิชา (§6)
- `ทหาร 1-4` + `รายวิชา|ผู้สอน` → สรุปผู้สอนรายชั้นปี (§6.3)
- `อัตลักษณ์`/`อัตลักษณ์ ปีN` + คอลัมน์ `ประเมิน งาม`/`ประเมิน สง่า` + sheet `ข้อคำถาม` → อัตลักษณ์ multi-dimension (§6ก)

## 8) Anti-pattern — สิ่งที่ห้ามผลิต (ตัวอย่างผิด)

ไฟล์บางไฟล์ใน Drive (เช่น `เจตคติ_...`, `นภาภิบาล_ข้อเสนอแนะ_...`) เป็น **output จากเครื่องมือเก่า**
ที่ **ทำข้อความคำถามหาย** — เป็นตัวอย่าง "ผิด" ที่ระบบใหม่ **ห้ามผลิตซ้ำ**:

```text
[X] sheet คำถาม ใช้ Q1 / คอลัมน์_4 / คอลัมน์_5 เป็น "ข้อความคำถาม"  → ผิด (ข้อความจริงหาย)
[X] มี SHA-256 / path เครื่อง (/Users/.../Downloads) ใน sheet แหล่งข้อมูล → นั่นคือของเครื่องมือเก่า
[X] มี sheet "สรุปตามชั้นปี" เป็น section หลัก → golden ไม่ทำ (แยกไฟล์/แยกวิชาแทน)
```

**กติกา:** ถ้าไฟล์ที่รับเข้ามามีแต่ `Q1..Qn / คอลัมน์_N` และ **ไม่มี sheet รายงานที่มีข้อความคำถามจริง**
→ **QA = REVIEW** และห้ามส่งเป็นไฟล์สมบูรณ์ (ต้องได้ข้อความคำถามจาก sheet รายงานก่อน)

## 9) อัลกอริทึมประมวลผล (สรุปให้ระบบทำตาม — กันมั่ว)

```text
1. เปิดไฟล์ → ดู sheet ทั้งหมด → จำแนกประเภท (§7):
   - เจอคู่ (ชมรมX)+X          → ประเภทกิจกรรม/ชมรม
   - เจอ sheet ต่อชื่ออาจารย์   → ประเภทผู้สอนรายวิชา
   - เจอ ทหาร 1-4 + รายวิชา|ผู้สอน → สรุปผู้สอนรายชั้นปี
2. หา "sheet รายงาน" ที่มี ข้อความคำถามจริง + เกณฑ์ 5 ระดับ
   - ถ้าไม่มี (มีแต่ Q1/คอลัมน์_N) → QA = REVIEW แล้วหยุด
3. อ่านข้อความคำถาม + เกณฑ์ จาก sheet รายงาน
4. อ่านคะแนนรายคน จาก sheet ดิบ (header อาจอยู่แถว 2, ยศแยกคอลัมน์, ข้อ1..n เป็นเลข)
5. คำนวณ X/SD รายข้อ (ทศนิยม 2), ค่าเฉลี่ยรวม, ร้อยละพึงพอใจ (สัดส่วน mean ≥ 3.51)
6. เติมผลลง "sheet รายงาน" ตาม layout เดิม (§2 / §6) — ไม่สร้าง layout ใหม่
7. PDF = พิมพ์ sheet รายงานตรง ๆ (A4, เกณฑ์+ตาราง+รวม+สรุป%)
8. QA gate: คะแนน 1-5, ไม่มีสูตร error, ข้อความคำถามเป็นข้อความจริง (ไม่ใช่ Q1/คอลัมน์_N)
```
