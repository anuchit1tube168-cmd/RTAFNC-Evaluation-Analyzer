# QA Acceptance Checklist — Excel/PDF Output

ใช้ตรวจรับไฟล์ที่ระบบ RTAFNC Evaluation Analyzer สร้างทุกครั้ง ก่อนนำไปใช้ทางราชการ

## 1) ตรวจเวอร์ชันระบบ

```text
[ ] Apps Script แสดง version v5 หรือสูงกว่า
[ ] Log ล่าสุดเป็น SUCCESS
[ ] ไม่มี Error: Drive is not defined
[ ] มี outputSpreadsheetUrl
[ ] มี pdfUrls ครบตามรายงานที่ต้องสร้าง
```

## 2) ตรวจ Excel / Google Sheet

### Sheet ที่ต้องมี

```text
[ ] Cover
[ ] Executive_Summary
[ ] Dashboard
[ ] Items_X_SD
[ ] Individual_All_Items
[ ] Comments_Themes ถ้ามีข้อคิดเห็น
[ ] QA_Log
[ ] Print_Report_รวม
[ ] Print_Report_ปี1
[ ] Print_Report_ปี2
[ ] Print_Report_ปี3
[ ] Print_Report_ปี4
```

### Clean Data

```text
[ ] ไม่มี Sheet1 เป็น output หลัก
[ ] ไม่มีชื่อไทยถูกตัดผิดช่อง
[ ] ไม่มีคอลัมน์ยศปลอมที่แยกตัวอักษรผิด
[ ] รหัส/เลขที่อ่านได้ถูกต้อง
[ ] ชั้นปีถูกต้อง
[ ] ตอบ/ทั้งหมดถูกต้อง
[ ] คะแนนอยู่ในช่วง 1–5
[ ] ค่า X ทศนิยม 2 ตำแหน่ง
[ ] ค่า SD ทศนิยม 2 ตำแหน่ง
[ ] ไม่มี #NAME?
[ ] ไม่มี #VALUE!
[ ] ไม่มี #DIV/0!
[ ] ไม่มี #REF!
[ ] ไม่มี #N/A
```

### Items_X_SD

```text
[ ] มีหัวข้อประเมินรายข้อครบ
[ ] มี N
[ ] มี X
[ ] มี SD
[ ] มีระดับ
[ ] มี sourceRow
[ ] หัวข้อยาว wrap text ไม่ถูกตัดทิ้ง
```

### Individual_All_Items

```text
[ ] มีข้อมูลรายบุคคลครบ
[ ] มีคะแนนรายข้อครบทุกข้อ
[ ] มี X รายบุคคล
[ ] มี SD รายบุคคล
[ ] มีระดับรายบุคคล
[ ] มีทะเบียน/ข้อคิดเห็น
```

## 3) ตรวจ PDF

### PDF ที่ต้องมี

```text

```

### โครงรายงาน

```text
[ ] หัวรายงานตรงกับ Excel
[ ] มีชื่อหน่วยงาน
[ ] มีหัวข้อรายงานพร้อมกลุ่มรายงาน
[ ] มีจำนวนผู้ตอบ
[ ] มีจำนวนข้อประเมิน
[ ] มี X รวม
[ ] มี SD รวม
[ ] มีเกณฑ์การประเมิน 5–1 ถ้าเป็นมาตราส่วน
[ ] มีตารางสรุปรายข้อ X/SD
[ ] มีข้อเสนอแนะการปรับปรุง
[ ] มีภาคผนวกรายบุคคล
[ ] มีหมายเหตุ QA
[ ] มีช่องลายเซ็น 3 ช่อง
```

### สิ่งที่ห้ามมี

```text
[ ] ไม่มี section สรุปตามชั้นปี
[ ] ไม่มีตารางขาดด้านขวา
[ ] ไม่มีหัวตารางเบี้ยว
[ ] ไม่มีตัวเลข X/SD หลุดคอลัมน์
[ ] ไม่มีข้อความทับกัน
[ ] ไม่มีหน้าเปล่าโดยไม่จำเป็น
[ ] ไม่มีชื่อถูกตัดผิดช่อง
```

## 4) เทียบ Golden Examples

```text
[ ] รูปแบบไม่ด้อยกว่ารายงานนภาภิบาลเดิม
[ ] รองรับรายงานรวม/รายชั้นปีแบบอาจารย์ที่ปรึกษาเดิม
[ ] รองรับข้อประเมินจำนวนมากแบบคุณลักษณะทางทหารเดิม
[ ] ถ้ามีข้อคิดเห็น ต้องสรุป theme/จำนวน/ร้อยละแบบรายงานข้อคิดเห็นเดิม
```

## 5) เกณฑ์ PASS / REVIEW

### PASS

ผ่านเมื่อครบทุกข้อหลัก:

```text
Excel clean
PDF print-safe
รายงานรวมและรายชั้นปีครบ
ไม่มีสูตร error
ไม่มีชื่อไทยผิดช่อง
มีข้อเสนอแนะ
มีลายเซ็น
มี QA_Log
```

### REVIEW

ตั้ง REVIEW เมื่อพบข้อใดข้อหนึ่ง:

```text
Drive is not defined
ไม่มี outputSpreadsheetUrl
ไม่มี PDF ครบ 5 ไฟล์
มี #NAME? หรือ formula error
PDF ขาดหน้า
ไม่มีข้อเสนอแนะ
ไม่มีลายเซ็น
มีสรุปตามชั้นปีใน PDF
ชื่อไทยถูกตัดผิดช่อง
หัวข้อรายงาน Excel/PDF ไม่ตรงกัน
```
