# Golden Examples — RTAFNC Evaluation Analyzer

เอกสารนี้ใช้เป็นแม่แบบอ้างอิงก่อนสร้างหรือแก้ระบบรายงาน Excel/PDF ทุกครั้ง

## หลักการ

ก่อนสร้าง output ใหม่ ระบบต้องเทียบกับไฟล์เก่าที่เคยทำสำเร็จแล้วอย่างน้อย 3 กลุ่ม:

1. รายงานนภาภิบาล
2. รายงานอาจารย์ที่ปรึกษา
3. รายงานคุณลักษณะทางทหาร/อัตลักษณ์
4. รายงานข้อคิดเห็น/ข้อเสนอแนะ

ระบบใหม่ต้องไม่ด้อยกว่าไฟล์เก่า และต้องเพิ่มข้อกำหนดใหม่ตาม SKILL.md v5.1

---

## 1) Golden Example: นภาภิบาล

ใช้สำหรับรายงานมาตราส่วน 1–5 ที่มีคะแนนรายข้อและรายบุคคล

### สิ่งที่ต้องคงไว้

```text
หัวรายงาน: รายงานผลการประเมิน ปีการศึกษา 2568
หน่วยงาน: วิทยาลัยพยาบาลทหารอากาศ
หัวข้อกิจกรรม: แบบประเมินกิจกรรม / ผลการประเมินคุณภาพกิจกรรม
เกณฑ์การประเมิน 5, 4, 3, 2, 1
ตาราง X / SD รายข้อ
ส่วนตรวจคุณภาพและแหล่งข้อมูล
ข้อมูลไฟล์ต้นทาง เช่น SHA-256
ภาคผนวกรายบุคคล
```

### สิ่งที่ต้องแก้เพิ่มจากตัวอย่างเดิม

```text
ตัด section สรุปตามชั้นปี ออกจาก PDF
สร้างไฟล์รวมชั้นปี 1-4
สร้างไฟล์แยกปี 1, ปี 2, ปี 3, ปี 4
เพิ่มข้อเสนอแนะการปรับปรุง
เพิ่มช่องลายเซ็นท้ายรายงาน
แก้ clean data ไม่ให้ชื่อไทยถูกตัดผิดช่อง
```

---

## 2) Golden Example: อาจารย์ที่ปรึกษา

ใช้สำหรับรายงานที่มีทั้งคะแนนและข้อคิดเห็น/ข้อเสนอแนะ

### สิ่งที่ต้องคงไว้

```text
รายงานรวมชั้นปี 1-4
รายงานแยกรายชั้นปี
เกณฑ์การประเมิน 5–1
ตาราง x / SD รายข้อ
ส่วนข้อคิดเห็นหรือข้อเสนอแนะเพิ่มเติม
สรุป Theme / จำนวน / ร้อยละ / ตัวอย่างข้อความ
```

### เมื่อตรวจพบข้อคิดเห็น

ต้องสร้าง sheet `Comments_Themes` และสรุป:

```text
ธีม | จำนวน | ร้อยละ | ตัวอย่างข้อความ | ข้อเสนอแนะเชิงปรับปรุง
```

ห้ามทิ้งข้อความข้อคิดเห็น

---

## 3) Golden Example: คุณลักษณะทางทหารและอัตลักษณ์

ใช้สำหรับรายงานที่มีจำนวนข้อเยอะและข้อความราชการยาว

### สิ่งที่ต้องคงไว้

```text
รายการประเมินจำนวนมาก
หัวข้อประเมินรายข้อพร้อม X / SD
ข้อความราชการครบถ้วน
ตารางไม่บีบจนอ่านไม่ได้
```

### กติกา layout

```text
Excel เก็บข้อความเต็ม
PDF ใช้ wrap text
คอลัมน์หัวข้อประเมินต้องกว้างที่สุด
X / SD ต้องอยู่กึ่งกลางและทศนิยม 2 ตำแหน่ง
```

---

## 4) Golden Example: รายงานข้อคิดเห็น

ใช้สำหรับไฟล์ที่มีข้อมูลแบบข้อคิดเห็น หรือคะแนนเป็น 0/0 แต่มีข้อความตอบกลับ

### Output ที่ต้องมี

```text
จำนวนผู้ตอบ
สรุปข้อคิดเห็น
Theme
จำนวน
ร้อยละ
ตัวอย่างข้อความ
ข้อเสนอแนะเชิงปรับปรุง
```

รายบุคคลให้แสดงเท่าที่จำเป็น และต้องไม่เปิดเผยข้อมูลเกินจำเป็น

## 5) Golden Example Comparison Gate

ก่อนถือว่า output ผ่าน ต้องเช็ก:

```text
[ ] หัวรายงานชัดเจนเท่ากับไฟล์เก่า
[ ] มีเกณฑ์ 5–1 ถ้าเป็นคะแนนมาตราส่วน
[ ] มี X / SD รายข้อ
[ ] มีแหล่งข้อมูลและ QA
[ ] มีภาคผนวกรายบุคคล
[ ] ถ้ามีข้อคิดเห็น ต้องมี Theme / จำนวน / ร้อยละ
[ ] รองรับรายงานรวมและรายชั้นปี
[ ] ไม่มี section สรุปตามชั้นปีใน PDF ใหม่
[ ] มีข้อเสนอแนะการปรับปรุง
[ ] มีช่องลายเซ็น
[ ] PDF พิมพ์แล้วไม่ขาด
```

ถ้าข้อใดไม่ผ่าน ให้ตั้งสถานะ `REVIEW` ไม่ใช่ `PASS`

ใช้ prompt นี้ทุกครั้งก่อนแก้ระบบ:

```text
ก่อนแก้ระบบ RTAFNC Evaluation Analyzer ให้ศึกษา SKILL.md v5.1 และ docs/GOLDEN_EXAMPLES.md ก่อน แล้วจึงแก้ Code.gs โดยรักษารูปแบบที่ดีจาก Golden Examples และเพิ่มข้อกำหนดใหม่: ตัดสรุปตามชั้นปีออก, สร้างรายงานรวมและแยกรายชั้นปี, เพิ่มข้อเสนอแนะ, เพิ่มลายเซ็น, และต้องผ่าน QA Gate ก่อน export PDF
```
