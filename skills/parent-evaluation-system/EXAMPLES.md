# Examples — Parent Evaluation System

## กิจกรรมที่ถูกต้อง

ปีการศึกษา 2569 ชื่อกิจกรรม “ประชุมผู้ปกครองประจำปี” และใช้ข้อคำถามจริง เช่น:

1. ความเหมาะสมของระยะเวลาในการจัดกิจกรรม
2. ความชัดเจนของข้อมูลที่วิทยาลัยสื่อสาร
3. ประโยชน์ที่ผู้ปกครองได้รับจากกิจกรรม

ผลที่คาดหวัง: สร้างกิจกรรมสำเร็จและ Item Dictionary มี 3 ข้อ

## Template ที่ต้องปฏิเสธ

- Q1
- Q2
- Column 4
- เลขล้วนโดยไม่มีข้อความคำถาม

ผลที่คาดหวัง: ระบบหยุดและให้สถานะ REVIEW

## บันทึกแบบประเมิน

ตัวอย่างข้อมูล:

- รหัสนักเรียน 001
- ชื่อ-สกุลนักเรียน นพอ. กนกวรรณ ธงสันเทียะ
- ชื่อผู้ปกครอง นายตัวอย่าง ผู้ปกครอง
- ความสัมพันธ์ บิดา
- คะแนน 5, 4, 4
- ข้อเสนอแนะ ควรเพิ่มเวลาถามตอบ

ผลที่คาดหวัง: ค่าเฉลี่ย 4.33 และบันทึกครบทุกข้อ

## คะแนนไม่ครบ

คะแนน 5, ว่าง, 4

ผลที่คาดหวัง: แจ้งข้อผิดพลาดและไม่เขียนข้อมูล

## Import Template ที่ถูกต้อง

หัวตาราง:

```text
ประทับเวลา | รหัสนักเรียน | ชื่อ-สกุลนักเรียน | ชื่อผู้ปกครอง |
ความสัมพันธ์ | ความเหมาะสมของกิจกรรม |
ความชัดเจนของการสื่อสาร | ประโยชน์ที่ได้รับ |
ค่าเฉลี่ย | S.D. | ข้อเสนอแนะ
```

ระบบต้องเลือกเฉพาะคอลัมน์คำถามจริง และตัดค่าเฉลี่ยกับ S.D.

## Import ซ้ำ

เมื่อนำเข้าชุดข้อมูลเดิมโดยไม่มีการเปลี่ยนแปลง ระบบต้องหยุด ไม่สร้างกิจกรรมหรือแถวคำตอบเพิ่ม และบันทึก receipt เดิมไว้ตรวจสอบ

## Dashboard

ผลลัพธ์ต้องมีจำนวนกิจกรรม จำนวนผู้ตอบ จำนวน PASS/REVIEW ค่าเฉลี่ย S.D. และผลรายข้อพร้อมข้อความคำถามจริง

## รายบุคคล

ใช้รหัสนักเรียนเป็นหลัก หากไม่มีรหัสให้ใช้ชื่อเต็มแบบตรงตัว ห้ามค้นด้วยชื่อบางส่วนเพราะอาจรวมคนละคน

## QA Receipt

```text
STATUS: REVIEW
SCOPE: Apps Script deployment
ROOT CAUSE: URL หลักเปิดหน้าระบบเดิม
CHANGES: เปลี่ยนเส้นทางหลักเป็น Modern Dashboard
TESTS: static PASS, runtime ยังไม่ยืนยัน
EVIDENCE: screenshot และ commit
RISK: ยังไม่มี workflow log
NEXT: ตรวจ Diagnostic บน URL จริง
```
