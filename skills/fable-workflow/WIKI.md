# 📘 Wiki: Fable Workflow — ติดตั้งกระบวนการคิดนี้ในทุกที่

SKILL.md ในชุดนี้คือ "กระบวนการคิด" ที่ถอดจากงานจริง (ระบบแบบประเมินนภาภิบาล)
เอาไปวางได้ 4 ที่ ต่างกันแค่วิธีติดตั้ง — เนื้อหาเดียวกัน

---

## 1. Claude Project (แนะนำสุด — ครอบทุกแชทใน project)

1. เปิด project → **Settings/⚙ → Project Instructions**
2. วางเนื้อหา SKILL.md ทั้งไฟล์ (ตัด frontmatter `---` หัวไฟล์ออกได้)
3. ทุกแชทใหม่ใน project จะคิดตามกระบวนการนี้อัตโนมัติ ไม่ต้องสั่งซ้ำ

**เคล็ดลับ:** สร้าง project ละโดเมน (สารบรรณ / OBE / งานปกครอง / Dev) แล้ววาง
Fable Workflow เป็น instructions ร่วม + เพิ่มกติกาเฉพาะโดเมนต่อท้าย เช่น
project สารบรรณเพิ่ม "เอกสารทุกฉบับตรวจ ตราครุฑ/เลขไทย/กั้นหน้า ตามระเบียบสารบรรณ"

## 2. Claude Code (สกิลถาวรในเครื่อง)

```bash
mkdir -p ~/.claude/skills/fable-workflow
cp SKILL.md ~/.claude/skills/fable-workflow/
```
Claude Code จะหยิบใช้เองเมื่องานเข้าเงื่อนไขใน description
(วางคู่กับ thai-docx, academic-paper-writer ที่มีอยู่ได้เลย ไม่ชนกัน)

## 3. Claude.ai แชทปกติ (User Preferences)

Preferences มีจำกัดพื้นที่ — ไม่ต้องวางทั้งไฟล์ ใส่แค่บรรทัดสรุป:
> "ทุกงานให้ทำตาม Fable Workflow: INSPECT ดูไฟล์จริงก่อน → CLARIFY ถามเฉพาะที่เปลี่ยนสถาปัตยกรรม → DESIGN บอกแนวทาง+trade-off+ข้อจำกัดก่อนสร้าง → BUILD ยืดหยุ่น/โคลนตัวเองได้/แก้ที่เดียวจบ → VERIFY บอกระดับที่ตรวจได้จริง → DELIVER พร้อม SKILL.md+WIKI.md+ข้อจำกัด+ขั้นถัดไป"

## 4. AI ตัวอื่น (ChatGPT, Gemini, ฯลฯ)

ใช้ได้เพราะเป็น markdown ล้วน ไม่ผูกกับ Claude:
- **ChatGPT:** วางใน Custom Instructions หรือสร้าง Custom GPT ใส่เป็น Instructions
- **Gemini:** วางใน Gems instructions
- **AI ใดๆ:** วางต้นบทสนทนา แล้วพิมพ์ "ทำงานตามกระบวนการนี้ทุกครั้ง"

ข้อจำกัดที่ต้องรู้: AI แต่ละตัวความสามารถต่างกัน — ตัวที่ไม่มี code execution
จะทำขั้น INSPECT/VERIFY ได้ไม่เต็ม (อ่านไฟล์เอง/รันทดสอบเองไม่ได้) กระบวนการยังใช้ได้
แต่ผลลัพธ์ขั้น 1 กับ 5 จะต้องให้คุณช่วยตรวจแทน

---

## ทำไม 6 ขั้นนี้ถึงเร็วและไม่หลุดรูปแบบ

| ขั้น | กันปัญหาอะไร |
|---|---|
| INSPECT | กันสร้างผิดทางเพราะเดาโครงสร้างข้อมูล (แพงที่สุดถ้าพลาด) |
| CLARIFY | กันถามพร่ำเพรื่อ และกันเดา requirement ที่เปลี่ยนสถาปัตยกรรม |
| DESIGN | กัน "ทำเสร็จแล้วใช้ไม่ได้" — ข้อจำกัดถูกบอกก่อนลงแรง |
| BUILD (โคลนตัวเองได้) | กันงานกลับมาหา AI ซ้ำทุกปี — ผลงานทำงานต่อเองได้ |
| VERIFY | กันความมั่นใจเกินจริง — บอกชัดว่าอะไรทดสอบแล้ว/ยัง |
| DELIVER สองชั้น | SKILL.md ให้ AI ตัวถัดไปทำงานต่อได้ทันที / WIKI ให้เพื่อนร่วมงานใช้ได้โดยไม่ต้องพึ่งคุณ |

หัวใจที่ทำให้ "AI ไหนก็ฉลาดขึ้น": ผลงานทุกชิ้นทิ้ง SKILL.md ไว้
→ ครั้งหน้าเปิดงานเดิม AI อ่าน SKILL.md แล้วรู้ invariants ทันที ไม่ต้องเริ่มจากศูนย์
→ ความรู้สะสมอยู่ที่ **ไฟล์** ไม่ใช่ที่ความจำของ AI ตัวใดตัวหนึ่ง

---

## โครงไฟล์แนะนำต่อ 1 โครงการ

```
โครงการ-xxx/
├── SKILL.md      ← AI อ่าน: สถาปัตยกรรม, invariants, จุดแก้ได้, debug
├── WIKI.md       ← คนอ่าน: ติดตั้ง, ใช้งานประจำ, แก้ปัญหา
├── assets/       ← ไฟล์ใช้งานจริง (template, .bas, ฯลฯ)
└── scripts/      ← สคริปต์สร้าง/ทดสอบ (build ใหม่จากศูนย์ได้เสมอ)
```

*ถอดแบบจากโครงการ napapiban-eval — ใช้เป็นตัวอย่างอ้างอิงโครงสร้างได้*
