# ระบบแปลผลการประเมินแผนกปกครอง วพอ.พอ.

> กฎเหล็กที่เรียนจากของจริง: **แต่ละหมวด/ปี format ไม่เหมือนกัน** อย่าใช้ตรรกะเดียวกับทุกไฟล์
> มีไฟล์ตัวอย่างสมบูรณ์ ~100 ไฟล์ ต้องเรียนจากไฟล์จริงก่อนเขียน ห้ามเดาโครงสร้าง

## กฎเหล็ก 5 ข้อ (อ่านทุกครั้ง)
1. เปิดไฟล์จริงก่อนเขียนโค้ด ดู header, คอลัมน์, ชั้นปีอยู่ที่ไหน, มี X/SD มาแล้วหรือยัง
2. Likert 5 ระดับ -> รายงาน x-bar + S.D. + แปลผล (ไม่ใช่คะแนนรวม/ผ่าน-ไม่ผ่าน)
3. คำนวณ S.D. แบบ STDEVP (ประชากร) ให้ตรงกับไฟล์ตัวอย่าง
4. PDF ฟอนต์ไทยผ่าน TH Sarabun PSK (ติดตั้งจริง) สร้าง HTML A4 แล้ว Print->PDF บนเว็บ
5. ทดสอบด้วยไฟล์จริงทุกครั้ง เทียบ x-bar รวม/แยกชั้นปี/จำนวนข้อ กับรายงานเดิม

## ตระกูล FORMAT ที่ยืนยันจากไฟล์จริงแล้ว (สำคัญที่สุด)
ระบบต้อง auto-detect ตระกูล จากโครงไฟล์ แล้วเลือก parser + template ให้ถูก

### ตระกูล A: กิจกรรม-รายด้าน TQF  [ทำเสร็จแล้ว]
- หมวด: นภาภิบาล, เจตคติ(บางแบบ)
- โครง: ชั้นปีแยกเป็น sheet (ปี4/ปี3/ปี2/ปี1), คอลัมน์ = เลขที่ ชื่อ email timestamp คะแนนรายข้อ
- ข้อ: 44 ข้อ มีข้อแม่/ข้อย่อย (6.1 > 6.1.1) จัดกลุ่มเป็น 8 ด้าน TQF
- รายงาน: ตาราง ด้าน|รายการ|x-bar|S.D. (merge ด้าน) + เกณฑ์ + ค่าเฉลี่ยรวม
- mapping: ดึงจากชีต "รายบุคคล 1-4" ของไฟล์ตัวอย่าง (dev/fmt_tqf.json)
- ตัวตรวจ: 44 ข้อ leaf 41 (ตัด 6.1/6.2/6.3) แต่ตารางแบบตัวอย่างแสดงครบ 44

### ตระกูล B: คะแนนแบนราบ + ชั้นปีในคอลัมน์  [ยังไม่รองรับ]
- หมวด: ศิลปวัฒนธรรม(ชมรม), น่าจะรวมจิตอาสา/กีฬา
- โครง: sheet เดียว, คอลัมน์ = ลำดับ คำนำหน้า ชื่อ "นพอ.ชั้นปีที่"(คอลัมน์) ข้อ1..20 X SD
- ต่างจาก A: ชั้นปีอยู่ในคอลัมน์, มี X/SD มาแล้วในไฟล์, ไม่มีด้าน TQF
- engine ปัจจุบันจับไม่ได้ (คืน group ว่าง) ต้องเพิ่ม parser สาขานี้

### ตระกูล C: ผู้สอน/ครูฝึกรายบุคคล  [ยังไม่สำรวจ]
- หมวด: ผู้สอนรายบุคคล วิชาการฝึกทหาร 1/2/3 ประเมินผู้สอนแยกคน

### ตระกูล D: สมรรถนะตาม LO/มคอ.  [ยังไม่สำรวจ]
- หมวด: แบบประเมินสมรรถนะผู้เรียนรายชั้นปีตาม LO (PLO/CLO)

### ตระกูล E: อาจารย์ที่ปรึกษา  [ทำเสร็จแล้ว 11 ข้อ]
- ชั้นปีแยก sheet, 11 ข้อ ไม่มีด้าน, มีคอลัมน์ comment ปลายเปิด

## กติกา AUTO-DETECT ตระกูล (เรียงลำดับตัดสิน)
1. มีคอลัมน์ header ตรง "ชั้นปี"/"นพอ.ชั้นปีที่" -> ตระกูล B
2. มีหลาย sheet ชื่อ ปี4/ปี3/ปี2/ปี1 -> ชั้นปีแยก sheet (A/E)
3. ข้อ >= 40 + มีรหัสข้อย่อย (x.y.z) -> ตระกูล A (TQF 8 ด้าน)
4. ข้อ ~11 + มี comment -> ตระกูล E
5. ชื่อไฟล์มี "ผู้สอน"/"ครูฝึก" -> ตระกูล C
6. ชื่อไฟล์มี "สมรรถนะ"/"LO"/"มคอ" -> ตระกูล D
- เมื่อไม่ชัด: เสนอผล detect ให้คนยืนยัน 1 คลิก ห้ามเดาเงียบ

## เกณฑ์แปลผล (Likert 5)
4.50-5.00 มากที่สุด | 3.50-4.49 มาก | 2.50-3.49 ปานกลาง | 1.50-2.49 น้อย | 1.00-1.49 น้อยที่สุด

## ไฟล์ในระบบ
- index.html: หน้าเว็บ (engine + SheetJS ฝังในไฟล์ + ปุ่มดาวน์โหลด/พิมพ์/บันทึก Drive)
- dev/engine.js: parser + คำนวณ (แก้ที่นี่)
- dev/shell.html + dev/build.py: ประกอบ index.html
- dev/report_exact.py: รายงานตามแบบตัวอย่าง (ตระกูล A)
- dev/fmt_tqf.json: mapping ด้าน+ข้อความข้อ ตระกูล A
- gas/uploader.gs: Apps Script เก็บไฟล์เข้า Drive
- .github/workflows/deploy-gas.yml: auto-deploy .gs ผ่าน clasp

## แผนขยาย (ทีละตระกูล ยืนยันกับผู้ใช้แล้ว)
โฟกัส 1 ตระกูล/ครั้ง ทำให้สมบูรณ์ก่อนขยาย
ลำดับ: A(เสร็จ) -> B(ศิลปวัฒนธรรม) -> C(ผู้สอน) -> D(LO)
เพิ่มตระกูลใหม่ = เพิ่ม branch ใน parseWorkbook + template + entry ใน auto-detect
