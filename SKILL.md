# SKILL.md — RTAFNC Evaluation Analyzer v5

มาตรฐานการสั่งงาน พัฒนา และตรวจรับระบบประมวลผลแบบประเมิน วิทยาลัยพยาบาลทหารอากาศ (วพอ.พอ.)

เอกสารนี้เป็นกติกาหลักของระบบ ตั้งแต่การ clean data ใน Excel การสร้างรายงาน PDF การ export รายชั้นปี และการตรวจคุณภาพก่อนนำไปใช้ทางราชการ

---

## 1) เป้าหมายระบบ v5

ระบบต้องทำงานได้ครบวงจร:

```text
อัปโหลดไฟล์ดิบ
→ เก็บใน Google Drive
→ แปลงเป็น Google Sheets
→ clean data
→ จำแนกหมวดงาน
→ ตรวจหัวข้อประเมินรายข้อ
→ ตรวจผู้ตอบและชั้นปี
→ คำนวณ X / SD ทศนิยม 2 ตำแหน่ง
→ สร้าง Excel รายงานรวมและแยกชั้นปี
→ Export PDF รวมและแยกชั้นปี
→ เพิ่มข้อเสนอแนะการปรับปรุง
→ เพิ่มช่องลายเซ็น
→ บันทึก QA Log
```

หลักสำคัญ:

1. Excel และ PDF ต้องมีหัวข้อรายงานตรงกันทุกไฟล์
2. ต้องมีไฟล์รวมชั้นปีที่ 1-4 และไฟล์แยกรายชั้นปี ปี 1, ปี 2, ปี 3, ปี 4
3. ห้ามมีสูตรเสีย เช่น `#NAME?`, `#VALUE!`, `#DIV/0!`
4. PDF ต้องพิมพ์ได้จริง ไม่ขาดหน้า ไม่เบี้ยว ไม่ตัดคอลัมน์
5. PDF ต้องมีข้อเสนอแนะการปรับปรุงและช่องลายเซ็น
6. ต้องตัดส่วน `สรุปตามชั้นปี` ออกจาก PDF ตามคำสั่งล่าสุด

## 2) Output ที่ต้องมี

### 2.1 Excel / Google Sheet

ไฟล์ Excel/Google Sheet ที่ผ่านมาตรฐานต้องมี sheet ต่อไปนี้:

```text
Cover
Executive_Summary
Dashboard
Items_X_SD
Individual_All_Items
QA_Log
Print_Report_รวม
Print_Report_ปี1
Print_Report_ปี2
Print_Report_ปี3
Print_Report_ปี4
```

ห้ามใช้ชื่อ sheet กำกวม เช่น `Sheet1`, `รวม`, `ปี4` เพียงอย่างเดียวใน output สุดท้าย หากต้องเก็บต้นทางไว้ให้เก็บใน sheet แยกชื่อ `Raw_*` หรือ `Source_*`

### 2.2 PDF

ต้อง export PDF อย่างน้อย 5 ไฟล์:

```text
รายงานรวมชั้นปีที่ 1-4.pdf
รายงานชั้นปีที่ 1.pdf
รายงานชั้นปีที่ 2.pdf
รายงานชั้นปีที่ 3.pdf
รายงานชั้นปีที่ 4.pdf
```

PDF ทุกไฟล์ต้อง export จาก sheet `Print_Report_*` เท่านั้น ไม่ export ทุก sheet รวมกัน

## 3) สิ่งที่ต้องตัดออก

ตัดส่วนนี้ออกจากรายงาน PDF:

```text
สรุปตามชั้นปี
```

เหตุผล: มีไฟล์แยกรายชั้นปีแล้ว จึงไม่ต้องมีตารางสรุปตามชั้นปีในรายงาน PDF หลัก เพื่อให้รายงานกระชับและไม่ซ้ำซ้อน

ถ้าต้องการเก็บข้อมูลชั้นปีเพื่อ QA ให้เก็บได้ใน `Dashboard` หรือ `QA_Log` แต่ห้ามแสดงเป็น section หลักใน PDF

## 4) มาตรฐานหัวข้อรายงาน

หัวข้อรายงานต้องตรงกันทั้ง Excel และ PDF

ให้ใช้ตัวแปรเดียวกัน เช่น:

```text
REPORT_TITLE
```

ตัวอย่างหัวข้อ:

```text
ผลการประเมินคุณภาพกิจกรรม การฝึกทหารร่วม ตามแผนนภาภิบาล : การฝึกร่วม 4 ชั้นปี ปีการศึกษา 2568 วันที่ 18 เมษายน 2568 ณ ศทย.อย.
```

การแสดงผล:

```text
Cover: หัวข้อรายงาน = REPORT_TITLE
Executive_Summary: หัวข้อรายงาน = REPORT_TITLE
Dashboard: หัวรายงาน = REPORT_TITLE
Print_Report_รวม: REPORT_TITLE — รวมชั้นปีที่ 1-4
Print_Report_ปี1: REPORT_TITLE — ชั้นปีที่ 1
Print_Report_ปี2: REPORT_TITLE — ชั้นปีที่ 2
Print_Report_ปี3: REPORT_TITLE — ชั้นปีที่ 3
Print_Report_ปี4: REPORT_TITLE — ชั้นปีที่ 4
```

## 5) มาตรฐาน Clean Data ใน Excel

### 5.1 ต้องตรวจและจัดข้อมูลผู้ตอบ

คอลัมน์มาตรฐานของผู้ตอบ:

```text
ลำดับ
รหัส/เลขที่
ชื่อ-สกุล
ชั้นปี
อีเมล
ตอบ/ทั้งหมด
ข้อ 1
ข้อ 2
...
ข้อ n
X
SD
ระดับ
ทะเบียน
ข้อคิดเห็น
```

### 5.2 ห้ามตัดตัวอักษรไทยผิดช่อง

ตัวอย่างปัญหาที่ต้องห้าม:

```text
ยศ = จ
ชื่อ = ชกร รัตนเวชสิทธิ
```

ต้อง clean ให้เป็น:

```text
ชื่อ-สกุล = กชกร รัตนเวชสิทธิ
```

ถ้าไม่มีข้อมูลยศ ไม่ต้องสร้างคอลัมน์ยศแยกเอง

### 5.3 สูตรต้องไม่เสีย

ห้ามมีค่าเหล่านี้ใน output สุดท้าย:

```text
#NAME?
#VALUE!
#DIV/0!
#REF!
#N/A
```

หากพบให้ถือว่า QA = REVIEW และห้าม export PDF จนกว่าจะแก้

### 5.4 คะแนนที่รับได้

รับเฉพาะคะแนน 1-5:

```text
1, 2, 3, 4, 5
```

หรือข้อความตัวเลข เช่น:

```text
5.0
4.0
```

ถ้าพบคะแนนมากกว่า 5 หรือต่ำกว่า 1 ให้บันทึกใน `QA_Log`

## 6) มาตรฐาน Items_X_SD

ต้องมีหัวข้อประเมินรายข้อครบ ไม่ใช่แค่ค่าเฉลี่ย

คอลัมน์บังคับ:

```text
ลำดับ | รหัสข้อ | หัวข้อประเมิน | N | X | SD | ระดับ | sourceRow
```

ตัวอย่าง:

```text
1 | 1.1 | สนใจ เอาใจใส่การเรียน การงาน หรือหน้าที่ที่ได้รับมอบหมาย | 251 | 4.53 | 0.67 | มากที่สุด | 1
```

กติกา:

1. หัวข้อประเมินต้องเป็นข้อความเต็มเท่าที่จัดหน้าได้
2. ถ้ายาวมากให้ wrap text ห้ามตัดข้อมูลทิ้งใน Excel
3. PDF ให้แสดงหัวข้อประเมินแบบอ่านได้ ไม่เบียดกับค่า X/SD
4. X และ SD ต้องเป็นทศนิยม 2 ตำแหน่ง

## 7) มาตรฐานรายบุคคล

รายบุคคลต้องมีทั้งคะแนนรายข้อและค่าเฉลี่ยรวม

### 7.1 Excel รายบุคคล

Sheet `Individual_All_Items` ต้องมีคอลัมน์:

```text
ลำดับ
รหัส/เลขที่
ชื่อ-สกุล
ชั้นปี
ตอบ/ทั้งหมด
ข้อ 1 + หัวข้อย่อ
ข้อ 2 + หัวข้อย่อ
...
ข้อ n + หัวข้อย่อ
X
SD
ระดับ
ทะเบียน
ข้อคิดเห็น
```

### 7.2 PDF รายบุคคล

PDF ให้มี section:

```text
ภาคผนวก: สรุปรายบุคคล
```

คอลัมน์ใน PDF:

```text
ลำดับ | รหัส | ชื่อ-สกุล | ชั้นปี | ตอบ/ทั้งหมด | X | ระดับ | ข้อคิดเห็น
```

ห้ามเอาคะแนนรายข้อทั้งหมด 44 ข้อไปใส่ใน PDF รายบุคคล เพราะจะกว้างเกินและทำให้พิมพ์ขาด ให้เก็บคะแนนรายข้อทั้งหมดใน Excel เท่านั้น

## 8) การแยกชั้นปี

ระบบต้องสร้างรายงาน 5 กลุ่ม:

```text
รวมชั้นปีที่ 1-4
ชั้นปีที่ 1
ชั้นปีที่ 2
ชั้นปีที่ 3
ชั้นปีที่ 4
```

### 8.1 วิธีจำแนกชั้นปี

ให้ใช้ข้อมูลตามลำดับความน่าเชื่อถือ:

1. คอลัมน์ `ชั้นปี`
2. ชื่อ sheet เช่น `ปี1`, `ปี 2`, `ชั้นปีที่ 3`
3. mapping จากรหัสนักเรียน/ทะเบียน ถ้ามี master registry
4. ถ้าไม่ทราบ ให้ใส่ `ตรวจสอบ` และบันทึก QA

### 8.2 ถ้าชั้นปีใดไม่มีข้อมูล

ยังต้องสร้าง sheet/PDF ของชั้นปีนั้น แต่ให้แสดง:

```text
ไม่พบข้อมูลผู้ตอบในชั้นปีนี้ กรุณาตรวจสอบไฟล์ต้นทางหรือ master registry
```

และให้ QA ของชั้นปีนั้นเป็น REVIEW

## 10) ข้อเสนอแนะการปรับปรุง

PDF ทุกไฟล์ต้องมี section:

```text
ข้อเสนอแนะการปรับปรุง
```

ระบบควรสร้างข้อเสนอแนะจาก 3 หัวข้อที่มีค่าเฉลี่ยต่ำสุด

ตัวอย่าง:

```text
1. ควรทบทวน/เสริมการดำเนินงานในหัวข้อ 2.3 สามารถจดจำความรู้ที่ได้รับอย่างครบถ้วน เนื่องจากค่าเฉลี่ยอยู่ที่ 4.26
2. ควรนำผลประเมินรายข้อไปกำหนดกิจกรรมปรับปรุงในรอบถัดไป และติดตามผลหลังดำเนินการ
3. ควรตรวจสอบ QA_Log และข้อมูลต้นทางก่อนนำรายงานไปใช้ทางราชการ
```

ถ้าคะแนนทุกข้ออยู่ในระดับดีมาก ให้ข้อเสนอแนะเป็นเชิงรักษามาตรฐานและต่อยอด

## 11) ลายเซ็นใน PDF

ท้าย PDF ทุกไฟล์ต้องมีช่องลายเซ็น 3 ช่อง:

```text
ลงชื่อ ........................................
(ผู้จัดทำรายงาน)

ลงชื่อ ........................................
(ผู้ตรวจทาน)

ลงชื่อ ........................................
(ผู้อนุมัติ/ผู้บังคับบัญชา)
```

ตำแหน่งลายเซ็นควรจัดเป็น 3 คอลัมน์ในแนวนอน ถ้าหน้ากว้างไม่พอให้แยกเป็น 3 บรรทัด แต่ห้ามล้นหน้า
