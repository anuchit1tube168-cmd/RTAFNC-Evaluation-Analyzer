# SKILL.md — RTAFNC Evaluation Analyzer v5.1

มาตรฐานการสั่งงาน พัฒนา และตรวจรับระบบประมวลผลแบบประเมิน วิทยาลัยพยาบาลทหารอากาศ (วพอ.พอ.)

เอกสารนี้เป็นกติกาหลักของระบบ ตั้งแต่การ clean data ใน Excel การสร้างรายงาน PDF การ export รายชั้นปี และการตรวจคุณภาพก่อนนำไปใช้ทางราชการ โดยให้ระบบเรียนรู้จากไฟล์เก่าที่เคยทำสำเร็จแล้วเป็น `Golden Examples` ทุกครั้ง

---

## 1) หลักคิด v5.1

ระบบต้องทำงานได้ครบวงจร:

```text
อัปโหลดไฟล์ดิบ
→ เก็บใน Google Drive
→ แปลงเป็น Google Sheets
→ clean data
→ จำแนกหมวดงาน
→ ตรวจหัวข้อประเมินรายข้อ
→ ตรวจผู้ตอบและชั้นปี
→ คำนวณ X / SD ทศนิยม 2 ตำแหน่ง
→ สร้าง Excel รายงานรวมและแยกชั้นปี
→ Export PDF รวมและแยกชั้นปี
→ เพิ่มข้อเสนอแนะการปรับปรุง
→ เพิ่มช่องลายเซ็น
→ บันทึก QA Log
```

หลักสำคัญ:

1. Excel และ PDF ต้องมีหัวข้อรายงานตรงกันทุกไฟล์
2. ต้องมีรายงานรวมชั้นปีที่ 1–4 และรายงานแยกปี 1, ปี 2, ปี 3, ปี 4
3. ห้ามมีสูตรเสีย เช่น `#NAME?`, `#VALUE!`, `#DIV/0!`, `#REF!`, `#N/A`
4. PDF ต้องพิมพ์ได้จริง ไม่ขาดหน้า ไม่เบี้ยว ไม่ตัดคอลัมน์
5. PDF ต้องมีข้อเสนอแนะการปรับปรุงและช่องลายเซ็น
6. ต้องตัด section `สรุปตามชั้นปี` ออกจาก PDF เพราะมีไฟล์แยกรายชั้นปีแล้ว
7. ก่อนแก้ระบบทุกครั้งต้องเทียบกับ Golden Examples ในข้อ 2

## 2) Golden Examples ที่ต้องเรียนรู้

ใช้ไฟล์เก่าที่เคยทำสำเร็จเป็นแม่แบบตรวจรับ ไม่ใช่สร้างรูปแบบใหม่โดยไม่ดูตัวอย่าง

### 2.1 Golden Example A — นภาภิบาล

ใช้เป็นแม่แบบสำหรับรายงานมาตราส่วน 1–5 ที่มีคะแนนรายข้อและรายบุคคล

ลักษณะที่ต้องเรียนรู้:

```text
หัวรายงาน: รายงานผลการประเมิน ปีการศึกษา 2568
หน่วยงาน: วิทยาลัยพยาบาลทหารอากาศ
หัวข้อกิจกรรม: แบบประเมินกิจกรรม/ผลการประเมินคุณภาพกิจกรรม
มีเกณฑ์ 5, 4, 3, 2, 1
มีตาราง X / SD รายข้อ
มีการตรวจคุณภาพและแหล่งข้อมูล
มี SHA-256 หรือข้อมูลไฟล์ต้นทาง
มีภาคผนวกรายบุคคล
```

จุดที่ต้องปรับปรุงจากตัวอย่างเดิม:

```text
ตัดตารางสรุปตามชั้นปีออกจาก PDF
แยก export เป็น รวมชั้นปี 1-4, ปี 1, ปี 2, ปี 3, ปี 4
เพิ่มข้อเสนอแนะการปรับปรุง
เพิ่มช่องลายเซ็นท้ายรายงาน
ป้องกันชื่อ/ยศถูกแยกผิด เช่น กชกร ถูกตัดเป็น จ + ชกร
```

### 2.2 Golden Example B — อาจารย์ที่ปรึกษา

ใช้เป็นแม่แบบสำหรับงานที่มีทั้งคะแนน 1–5 และข้อคิดเห็น/ข้อเสนอแนะ

ลักษณะที่ต้องเรียนรู้:

```text
รายงานรวมชั้นปีที่ 1-4
รายงานแยกรายชั้นปี เช่น ปี 1, ปี 2, ปี 4
มีเกณฑ์ 5–1
มีตาราง x / SD รายข้อ
มีส่วนข้อคิดเห็นหรือข้อเสนอแนะเพิ่มเติม
มีสรุป theme / จำนวน / ร้อยละ / ตัวอย่างข้อความ
```

กติกาเมื่อเจอข้อมูลข้อคิดเห็น:

```text
ห้ามทิ้งข้อความข้อเสนอแนะ
ต้องสรุปเป็น theme
ต้องนับจำนวนและร้อยละ
ต้องเก็บตัวอย่างข้อความที่ไม่ระบุตัวตน
ต้องมีข้อเสนอแนะเชิงปรับปรุงใน PDF
```

### 2.3 Golden Example C — คุณลักษณะทางทหารและอัตลักษณ์

ใช้เป็นแม่แบบสำหรับแบบประเมินที่มีจำนวนข้อมาก และมีประเด็นวินัย/แบบธรรมเนียมทหาร/ราชการ

ลักษณะที่ต้องเรียนรู้:

```text
มีรายการประเมินจำนวนมาก
มีหัวข้อประเมินรายข้อพร้อม X / SD
ข้อความรายข้ออาจยาว ต้อง wrap text
ต้องรักษาข้อความราชการให้ครบถ้วน ไม่ตัดคำสำคัญ
```

### 2.4 Golden Example D — รายงานข้อคิดเห็น

ใช้เป็นแม่แบบสำหรับไฟล์ที่ไม่มีคะแนน หรือคะแนนเป็น 0/0 แต่มีข้อความข้อเสนอแนะ

ลักษณะที่ต้องเรียนรู้:

```text
รูปแบบข้อมูล = ข้อคิดเห็น
จำนวนผู้ตอบ
สรุปข้อคิดเห็น
ธีม | จำนวน | ร้อยละ | ตัวอย่าง
รายบุคคลให้แสดงเท่าที่จำเป็น ไม่เปิดเผยข้อมูลเกินจำเป็น
```

ต้อง export PDF อย่างน้อย 5 ไฟล์:


## 4) สิ่งที่ต้องตัดออกจาก PDF

ตัด section นี้ออกจากรายงาน PDF:

```text
สรุปตามชั้นปี
```

เหตุผล: มีไฟล์แยกรายชั้นปีแล้ว จึงไม่ต้องมีตารางสรุปตามชั้นปีในรายงาน PDF หลัก เพื่อให้รายงานกระชับและไม่ซ้ำซ้อน

อนุญาตให้เก็บสถิติชั้นปีไว้ใน `Dashboard` หรือ `QA_Log` เพื่อการตรวจสอบเท่านั้น แต่ห้ามแสดงเป็น section หลักใน PDF

## 5) มาตรฐานหัวข้อรายงาน

หัวข้อรายงานต้องตรงกันทั้ง Excel และ PDF ให้ใช้ตัวแปรเดียวกัน:

```text
REPORT_TITLE
```

ตัวอย่างหัวข้อ:

```text
ผลการประเมินคุณภาพกิจกรรม การฝึกทหารร่วม ตามแผนนภาภิบาล : การฝึกร่วม 4 ชั้นปี ปีการศึกษา 2568 วันที่ 18 เมษายน 2568 ณ ศทย.อย.
```

การแสดงผล:

```text
Cover: หัวข้อรายงาน = REPORT_TITLE
Executive_Summary: หัวข้อรายงาน = REPORT_TITLE
Dashboard: หัวรายงาน = REPORT_TITLE
Print_Report_รวม: REPORT_TITLE — รวมชั้นปีที่ 1-4
Print_Report_ปี1: REPORT_TITLE — ชั้นปีที่ 1
Print_Report_ปี2: REPORT_TITLE — ชั้นปีที่ 2
Print_Report_ปี3: REPORT_TITLE — ชั้นปีที่ 3
Print_Report_ปี4: REPORT_TITLE — ชั้นปีที่ 4
PDF file name: ต้องมีหมวด + ชั้นปี + ปีการศึกษา
```

## 6) มาตรฐาน Clean Data ใน Excel

### 6.1 ข้อมูลผู้ตอบ

คอลัมน์มาตรฐานของผู้ตอบ:

```text
ลำดับ
รหัส/เลขที่
ชื่อ-สกุล
ชั้นปี
อีเมล
ตอบ/ทั้งหมด
ข้อ 1
ข้อ 2
...
ข้อ n
X
SD
ระดับ
ทะเบียน
ข้อคิดเห็น
```

### 6.2 ห้ามตัดตัวอักษรไทยผิดช่อง

ตัวอย่างปัญหาที่ต้องห้าม:

```text
ยศ = จ
ชื่อ = ชกร รัตนเวชสิทธิ
```

ต้อง clean ให้เป็น:

```text
ชื่อ-สกุล = กชกร รัตนเวชสิทธิ
```

ถ้าไม่มีข้อมูลยศ ไม่ต้องสร้างคอลัมน์ยศแยกเอง

### 6.3 สูตรต้องไม่เสีย

ห้ามมีค่าเหล่านี้ใน output สุดท้าย:

```text
#NAME?
#VALUE!
#DIV/0!
#REF!
#N/A
```

หากพบให้ถือว่า QA = REVIEW และห้าม export PDF จนกว่าจะแก้

### 6.4 คะแนนที่รับได้

รับเฉพาะคะแนน 1–5:

```text
1, 2, 3, 4, 5
```

หรือข้อความตัวเลข เช่น `5.0`, `4.0`

ถ้าพบคะแนนมากกว่า 5 หรือต่ำกว่า 1 ให้บันทึกใน `QA_Log`

## 7) มาตรฐาน Items_X_SD

ต้องมีหัวข้อประเมินรายข้อครบ ไม่ใช่แค่ค่าเฉลี่ย

คอลัมน์บังคับ:

```text
ลำดับ | รหัสข้อ | หัวข้อประเมิน | N | X | SD | ระดับ | sourceRow
```

ตัวอย่าง:

```text
1 | 1.1 | สนใจ เอาใจใส่การเรียน การงาน หรือหน้าที่ที่ได้รับมอบหมาย | 251 | 4.53 | 0.67 | มากที่สุด | 1
```

กติกา:

1. หัวข้อประเมินต้องเป็นข้อความเต็มเท่าที่จัดหน้าได้
2. ถ้ายาวมากให้ wrap text ห้ามตัดข้อมูลทิ้งใน Excel
3. PDF ให้แสดงหัวข้อประเมินแบบอ่านได้ ไม่เบียดกับค่า X/SD
4. X และ SD ต้องเป็นทศนิยม 2 ตำแหน่ง

## 8) มาตรฐานรายบุคคล

รายบุคคลต้องมีทั้งคะแนนรายข้อและค่าเฉลี่ยรวม

### 8.1 Excel รายบุคคล

Sheet `Individual_All_Items` ต้องมีคอลัมน์:

```text
ลำดับ
รหัส/เลขที่
ชื่อ-สกุล
ชั้นปี
ตอบ/ทั้งหมด
ข้อ 1 + หัวข้อย่อ
ข้อ 2 + หัวข้อย่อ
...
ข้อ n + หัวข้อย่อ
X
SD
ระดับ
ทะเบียน
ข้อคิดเห็น
```

### 8.2 PDF รายบุคคล

PDF ให้มี section:

```text
ภาคผนวก: สรุปรายบุคคล
```

คอลัมน์ใน PDF:

```text
ลำดับ | รหัส | ชื่อ-สกุล | ชั้นปี | ตอบ/ทั้งหมด | X | ระดับ | ข้อคิดเห็น
```

PDF ไม่ต้องแสดงคะแนนทุกข้อรายบุคคล เพราะจะกว้างเกินและพิมพ์ขาด ให้เก็บคะแนนรายข้อทั้งหมดใน Excel แทน

## 9) มาตรฐานรายงานข้อคิดเห็น

ถ้าไฟล์มีข้อคิดเห็นหรือข้อเสนอแนะ ต้องสร้าง sheet:

```text
Comments_Themes
```

คอลัมน์บังคับ:

```text
ธีม | จำนวน | ร้อยละ | ตัวอย่างข้อความ | ข้อเสนอแนะเชิงปรับปรุง
```

Theme เริ่มต้นที่ระบบควรรู้จัก:

```text
การให้คำปรึกษา/การบริการ
การสื่อสาร/การรับฟัง
ความใส่ใจ/ความเอาใจใส่
ความเป็นส่วนตัว/ความลับ
ความพร้อมของสถานที่/ระบบ
ข้อเสนอแนะอื่น ๆ
```

PDF ต้องมีสรุปข้อคิดเห็นใน Executive Summary หรือส่วนข้อเสนอแนะการปรับปรุง

## 10) โครงสร้าง PDF มาตรฐาน

PDF ทุกไฟล์ต้องมีลำดับ section ดังนี้:

```text
1. หัวรายงาน
2. หน่วยงาน
3. หัวข้อรายงาน + กลุ่มรายงาน เช่น รวมชั้นปีที่ 1-4 หรือ ชั้นปีที่ 2
4. ข้อมูลสรุป: จำนวนผู้ตอบ, จำนวนข้อ, X, SD, ระดับ
5. เกณฑ์การประเมิน 5–1
6. ตารางสรุปรายข้อประเมิน X / SD
7. ข้อเสนอแนะการปรับปรุง
8. ภาคผนวก: สรุปรายบุคคล
9. หมายเหตุการตรวจคุณภาพ
10. ช่องลายเซ็น
```

ห้ามใส่ section `สรุปตามชั้นปี`

## 11) ข้อเสนอแนะการปรับปรุง

PDF ต้องมีข้อเสนอแนะอย่างน้อย 3 ข้อ โดยสร้างจากข้อมูลจริง:

1. เลือกหัวข้อที่มีค่า X ต่ำที่สุด 3 อันดับแรก
2. เขียนข้อเสนอแนะให้สัมพันธ์กับหัวข้อนั้น
3. ถ้ามีข้อคิดเห็น ให้ใช้ theme ที่มีจำนวนมากที่สุดประกอบ
4. ห้ามเขียนข้อเสนอแนะแบบเดาโดยไม่มีข้อมูลรองรับ

รูปแบบ:

```text
ข้อเสนอแนะการปรับปรุง
1. ควรเสริมการดำเนินงานในหัวข้อ ... เนื่องจากมีค่าเฉลี่ย ...
2. ควรจัดกิจกรรม/มาตรการติดตามผลในหัวข้อ ...
3. ควรนำข้อคิดเห็น theme ... ไปใช้ในการปรับปรุงรอบถัดไป
```

## 12) ช่องลายเซ็น

ท้าย PDF ต้องมีช่องลายเซ็นอย่างน้อย 3 ช่อง:

```text
ลงชื่อ ........................................
(ผู้จัดทำรายงาน)

ลงชื่อ ........................................
(ผู้ตรวจทาน)

ลงชื่อ ........................................
(ผู้อนุมัติ/ผู้บังคับบัญชา)
```

ถ้าผู้ใช้ระบุชื่อ/ตำแหน่ง ให้ใส่ชื่อจริงแทนข้อความ placeholder ได้

## 13) Page Setup / Print Safety

ใช้ค่า export PDF:

```text
A4
landscape
fit width
hide gridlines
show page numbers
margin แคบ
export เฉพาะ Print_Report_* ด้วย gid
```

ค่าจัดหน้า:

```text
Font: TH Sarabun PSK หรือ TH Sarabun New
Header color: #0B2347
Accent: #D6A94A
Number format: 0.00
Wrap text: เปิดทุก cell ที่มีข้อความ
Alignment: หัวตารางกึ่งกลาง, ตัวเลขกึ่งกลาง, ข้อความซ้าย
Column item text: กว้างที่สุด
```

ถ้า PDF เกินหน้า:

1. ลดจำนวนรายบุคคลต่อหน้า หรือแยกภาคผนวก
2. ไม่แสดงคะแนนรายข้อรายบุคคลใน PDF
3. ให้เก็บคะแนนรายข้อเต็มใน Excel เท่านั้น
4. ห้ามบีบจนตัวหนังสืออ่านไม่ออก

## 15) Golden Example Comparison Checklist

```text

[ ] มีข้อมูลไฟล์ต้นทาง/QA
```


```text

[ ] ข้อความยาว wrap ได้

```

## 19) หมายเหตุสำหรับนักพัฒนา

1. ถ้าแก้ parser ให้ทดสอบกับ Golden Examples ก่อนเสมอ
2. ห้ามลบ logic ที่สร้างไฟล์แยกรายชั้นปี
3. ห้ามเอา `สรุปตามชั้นปี` กลับเข้า PDF
4. ห้าม export ทุก sheet เป็น PDF เดียว
5. ห้ามสร้างรายบุคคลใน PDF แบบกว้างจนตัดหน้า
6. ถ้าข้อมูลรายบุคคลยาว ให้ PDF แสดงสรุป ส่วน Excel เก็บรายละเอียดเต็ม
7. ถ้ามีข้อคิดเห็น ให้รักษา privacy และสรุปเป็น theme
8. ถ้าไม่มั่นใจ ให้สถานะ REVIEW ไม่ใช่ PASS
